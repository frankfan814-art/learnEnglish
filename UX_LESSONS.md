# UX 设计问题记录

## 问题记录

### 2025-01-17: 导航按钮顺序错误

**问题描述：**
导航控制按钮的顺序不符合用户使用习惯。

**错误代码：**
```jsx
// ❌ 错误：下一个在左边，上一个在右边
<button className="nav-button next-button">下一个 ▶</button>
<button className="nav-button prev-button">上一个 ◀</button>
```

**正确代码：**
```jsx
// ✅ 正确：上一个在左边，下一个在右边
<button className="nav-button prev-button">上一个 ◀</button>
<button className="nav-button next-button">下一个 ▶</button>
```

**原因分析：**
1. 用户习惯：← 上一个（左） | 下一个 → （右）
2. 这符合阅读方向（从左到右）
3. 符合大多数应用的导航模式（如翻页、音乐播放器等）

**修复位置：**
- `src/components/NavigationControls.jsx`

## 通用 UX 原则

### 导航控制

| 操作 | 按钮位置 | 图标 | 键盘快捷键 |
|------|---------|------|-----------|
| 上一个/前一个 | **左边** | ◀ ← | ← ArrowLeft |
| 下一个/后一个 | **右边** | ▶ → | → ArrowRight |

### 设计检查清单

- [ ] 确认按钮布局符合用户习惯
- [ ] 确认图标方向与按钮功能一致
- [ ] 确认键盘快捷键符合通用标准
- [ ] 确认移动端适配（拇指可操作区域）

## 其他需注意的问题

### 1. 表单填写方向
- 从上到下、从左到右填写
- Tab 键应按此顺序移动焦点

### 2. 列表滚动
- 下拉加载更多：向上滑动查看更早的内容
- 上拉刷新：向下滑动刷新

### 3. 模态框操作
- 主要操作（确认/保存）：右边或下方，视觉权重更高
- 次要操作（取消/关闭）：左边或上方，视觉权重较低

### 4. 颜色语义
- 绿色：成功、确认、前进
- 红色：删除、取消、警告
- 灰色：禁用、次要操作

---

## 避免错误的方法

1. **用户测试**：在开发早期进行用户测试
2. **竞品分析**：参考同类应用的交互模式
3. **设计规范**：参考 Material Design、Apple HIG 等设计规范
4. **代码审查**：PR 时检查 UX 相关代码
